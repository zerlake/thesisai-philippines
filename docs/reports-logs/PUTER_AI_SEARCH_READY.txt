================================================================================
HOW IT WORKS
================================================================================

SCENARIO 1: Puter Available
  1. User types query
  2. Clicks "AI Search" button
  3. Connects to Puter AI
  4. Returns AI-ranked papers
  5. Shows success toast

SCENARIO 2: Puter Unavailable
  1. Button detects Puter not available
  2. Button becomes disabled (visible but grayed)
  3. Tooltip: "Puter AI not available"
  4. User can click "Search" (standard)
  5. Gets results via APIs

SCENARIO 3: AI Search Fails
  1. User clicks "AI Search"
  2. Puter is available but request fails
  3. Shows error toast
  4. Auto-fallback to standard search
  5. User gets results anyway

TROUBLESHOOTING
================================================================================

Q: AI Search button is disabled?
A: Check if Puter.js is loaded in your layout/document

Q: No AI results appearing?
A: Try standard "Search" button as fallback

Q: No [PuterAI] logs?
A: 1. Check Puter.js loaded
   2. Check browser console filter
   3. Try different search query

Q: Fallback happening every time?
A: Normal if Puter unavailable in your environment
   Standard search will provide results

Q: How do I enable Puter?
A: 1. Add Puter.js Script to layout
   2. Ensure user is logged into Puter
   3. Verify Puter account is set up
